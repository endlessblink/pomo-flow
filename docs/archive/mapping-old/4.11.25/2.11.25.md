# Pomo-Flow Design Tokens Implementation Map

**Version**: 2.11.25
**Date**: November 2, 2025
**Project**: Pomo-Flow Vue 3 Productivity Application

---

## Executive Summary

Pomo-Flow implements a **world-class design tokens system** with 755 lines of comprehensive CSS custom properties, organized in a sophisticated 3-tier architecture. The system provides complete design consistency across the application while enabling powerful runtime theming and advanced glass morphism effects.

**Key Statistics:**
- **755 lines** of design tokens
- **200+ CSS custom properties**
- **3-tier token architecture** (Base → Semantic → Component)
- **593-line Tailwind configuration** with custom plugins
- **Complete dark/light theme support** with runtime switching

## 7. Theme Management System

### Reactive Theme Store
**File**: `src/stores/theme.ts`

#### Core Architecture
```typescript
export const useThemeStore = defineStore('theme', () => {
  const currentThemeId = ref<string>('dark')
  const currentTheme = ref<Theme>(themes['dark'])
  const isDarkMode = ref(currentThemeId.value === 'dark')
})
```

#### Theme Application
```typescript
const applyThemeTokens = (theme: Theme) => {
  const root = document.documentElement

  Object.entries(theme.tokens).forEach(([key, value]) => {
    root.style.setProperty(`--${key}`, value)
  })

  // Update dark-theme class for legacy styles
  root.classList.toggle('dark-theme', theme.id === 'dark')
}
```

## 9. Implementation Guidelines

### When to Use Which Tokens

#### **Surface Colors**
- **Primary**: Main application background
- **Secondary**: Card/panel backgrounds
- **Tertiary**: Elevated surfaces
- **Elevated**: Floating elements

#### **Text Colors**
- **Primary**: Main content, important text
- **Secondary**: Supporting text, descriptions
- **Tertiary**: Metadata, timestamps
- **Muted**: Disabled states, placeholders
- **Subtle**: Very low emphasis text

#### **Border Colors**
- **Subtle**: Default borders
- **Medium**: Hover states
- **Strong**: Focus states
- **Interactive**: Clickable elements

#### **State Colors**
- **Work**: Active Pomodoro sessions
- **Break**: Break periods
- **Focus**: Deep work sessions
- **Success**: Completed actions
- **Warning**: Caution states
- **Danger**: Delete actions

### Best Practices

#### **Token Usage**
```css
/* ✅ Good - Use semantic tokens */
.component {
  background: var(--surface-secondary);
  color: var(--text-primary);
  border: 1px solid var(--border-subtle);
}

/* ❌ Avoid - Hardcoded values */
.component {
  background: #2d3748;
  color: #f1f5f9;
  border: 1px solid rgba(255, 255, 255, 0.06);
}
```

#### **Glass Morphism**
```css
/* ✅ Good - Subtle glass effects */
.glass-element {
  background: var(--glass-bg-light);
  backdrop-filter: blur(var(--blur-sm));
  border: 1px solid var(--glass-border);
}

/* ❌ Avoid - Overly dramatic effects */
.glass-element {
  background: rgba(255, 255, 255, 0.2);
  backdrop-filter: blur(20px);
  border: 2px solid rgba(255, 255, 255, 0.5);
}
```

#### **Animation**
```css
/* ✅ Good - Use system animations */
.smooth-transition {
  transition: all var(--duration-normal) var(--spring-smooth);
}

/* ❌ Avoid - Hardcoded timings */
.smooth-transition {
  transition: all 200ms cubic-bezier(0.4, 0, 0.2, 1);
}
```

### Performance Optimization

#### **GPU Acceleration**
```css
.gpu-accelerated {
  transform: translateZ(0);
  will-change: transform;
}
```

#### **Reduced Motion Support**
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
